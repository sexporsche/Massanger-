# Prime-Media Messenger — руководство администратора и пользователя

Документ для **системных администраторов** (развёртывание сервера) и **сотрудников АО «Прайм-Медиа»** (ежедневная работа в приложении).

---

# Часть A. Для администратора

## Назначение

Раздел предназначен для специалистов, которые разворачивают и сопровождают **серверную часть** мессенджера и при необходимости обновляют клиентские сборки с новым TLS-сертификатом.

## Требования к серверу

| Требование | Описание |
|------------|----------|
| ОС | Linux (Ubuntu 20.04+), Windows Server 2019+ или macOS |
| Python | 3.8 или выше |
| OpenSSL | Для скрипта `generate_certs.sh` |
| Сеть | Доступный TCP-порт **12345** в локальной сети (без блокировки файрволом) |
| Диск | Место под `server/data/messenger.db` и `server/data/uploads/` |

## Установка и запуск сервера

1. Скопируйте каталог `server/` на целевую машину.

2. Установите зависимости (если добавлен `requirements.txt`):
   ```bash
   pip install -r requirements.txt
   ```
   При отсутствии файла достаточно стандартной установки Python 3.8+.

3. Перейдите в каталог сервера:
   ```bash
   cd server
   ```

4. Сгенерируйте TLS-сертификаты:
   ```bash
   chmod +x generate_certs.sh
   ./generate_certs.sh
   ```

5. Запустите сервер с отладочным логированием:
   ```bash
   python3 main.py -d
   ```
   Флаг `-d` пишет подробные логи в консоль и в `logs/messenger_YYYYMMDD.log`.

6. Обновите доверие на клиентах: скопируйте `certs/cert.pem` в  
   `app/src/main/res/raw/messenger_server_cert` и пересоберите APK (см. `DEVELOPER_GUIDE.md`).

7. Убедитесь, что в `app/src/main/res/values/strings.xml` на клиентах указан **IP сервера** в LAN.

## Управление базой данных сервера

| Объект | Путь / действие |
|--------|-----------------|
| Файл БД | `server/data/messenger.db` (SQLite) |
| Загрузки файлов | `server/data/uploads/` |
| Резервная копия | `python3 backup_db.py` (из каталога `server/`) |
| Пользователи мессенджера | Таблица `users` (логин, хеш пароля, соль); при первом запуске создаются `user1`–`user3` и `admin` |
| Пользователи Prime (профили в приложении) | Создаются **администратором** в клиенте (Admin Panel → Room) или через mock-регистрацию; на сервере Python отдельной таблицы профилей Prime нет |

### Рекомендации по эксплуатации

- Регулярно выполняйте `backup_db.py` перед обновлением сервера.
- Не публикуйте `certs/key.pem` и `messenger.db` в открытых репозиториях.
- Для production рассмотрите systemd-сервис и ротацию логов (см. `INSTALLATION.md` в репозитории, если присутствует).

### Роль ADMIN в приложении

Учётная запись с логином **`admin`** (пароль по умолчанию **`admin`** на тестовом сервере) получает роль **ADMIN** на клиенте. Администратор управляет профилями сотрудников в разделе **«Админ: пользователи»**, а не через SQL напрямую (рекомендуемый путь).

# Часть B. Для пользователя (сотрудника)

## Назначение

Раздел для сотрудников АО «Прайм-Медиа», использующих мобильное приложение **Prime-Media Messenger** для новостей, сообщений и корпоративных опросов.

## Вход в систему

1. Запустите приложение **Prime-Media Messenger** (иконка «Прайм-Медиа»).

2. Введите **корпоративный логин** и **пароль**, выданные администратором (на тестовом стенде: например, `user1` / `password1`).

3. Нажмите **«Войти»**.

4. При успешном входе откроется главный экран с нижней панелью: **Главная**, **Новости**, **Чаты**, **Алерты**, **Ещё**.

5. При ошибке логина или пароля останетесь на экране входа — проверьте данные и доступность сервера в сети.

## Основные возможности

### Лента новостей (вкладка «Новости»)

1. Откройте вкладку **«Новости»** внизу экрана.

2. Просматривайте корпоративные новости и объявления в списке карточек.

3. Нажмите на карточку, чтобы открыть **полный текст**, **комментарии** и кнопку **«Лайк»**.

4. Нажмите **«Лайк»**, чтобы увеличить счётчик реакций.

5. Закреплённые материалы отмечены чипом **«Закреплено»**.

### Личные сообщения (вкладка «Чаты» → «Личные»)

1. Откройте вкладку **«Чаты»**.

2. В верхней части выберите сегмент **«Личные»**.

3. Нажмите синюю кнопку **контактов** (FAB) в правом нижнем углу.

4. В списке выберите сотрудника.

5. Введите текст в поле сообщения и нажмите **«Отправить»**.

6. История подгружается с сервера и сохраняется локально; при кратковременном отсутствии сети видны ранее загруженные сообщения.

7. Для отправки **файла** используйте кнопку вложения в экране чата (поддерживаются изображения, PDF, документы Office, текст).

### Групповые чаты и каналы отделов (вкладка «Чаты»)

1. В разделе **«Чаты»** выберите **«Группы»** или **«Каналы»**.

2. Нажмите на нужный поток (например, **«Проект „Эфир“»**).

3. Откроется экран с **историей из локальной базы** (режим просмотра; отправка в группе может быть ограничена в текущей версии).

### Профиль (вкладка «Ещё» → «Профиль»)

1. Откройте **«Ещё»** → **«Профиль»**.

2. Просмотрите ФИО, должность, отдел, контакты, статус.

3. **Сотрудник** может изменить **статус** и **аватар** (кнопка смены аватара / поле статуса с сохранением).

4. Редактирование должности, отдела и роли через этот экран **недоступно** — только для администратора.

### Опросы (вкладка «Ещё» → «Опросы»)

1. Откройте **«Ещё»** → **«Опросы»**.

2. Прочитайте вопрос и варианты ответов.

3. Нажмите на вариант — голос учитывается, счётчик голосов обновляется.

### Уведомления (вкладка «Алерты»)

1. Откройте **«Алерты»**.

2. Просматривайте входящие корпоративные уведомления.

3. При необходимости нажмите **«Прочитать всё»**.

### Настройки и выход

1. Откройте **настройки** через иконку вверху или **«Ещё»** → **«Настройки»**.

2. Переключите **тёмную тему** и флаг push (локальная настройка интерфейса).

3. **«Выйти из аккаунта»** очищает сессию и возвращает на экран входа.

## Для руководителей и администраторов

### Создание новости (MANAGER, LEADER, ADMIN)

1. Перейдите на вкладку **«Новости»**.

2. Нажмите кнопку **«+»** (FAB) — она видна только при роли с правом публикации.

3. Заполните **заголовок** и **текст**.

4. При необходимости включите **«Закрепить»** или **«Объявление»**.

5. Нажмите **«Опубликовать»** — материал появится в ленте.

Сотрудник с ролью **EMPLOYEE** не видит FAB и пункт создания новости.

### Администрирование пользователей (только ADMIN)

1. Откройте **«Ещё»** → **«Админ: пользователи»**.

2. Используйте **поиск** по ФИО, email, телефону, должности, роли.

3. **Редактирование:** нажмите на пользователя в списке, измените поля, роль, отдел, при необходимости включите **«Заблокировать вход»**, нажмите **«Сохранить»**.

4. **Создание:** нажмите **«+»**, заполните форму нового сотрудника, нажмите **«Создать»**.

5. Просмотр чужого профиля: на экране профиля другого пользователя доступна кнопка **«Редактировать (админ)»**.
