# Prime-Media Messenger — установка и тестирование

Документ описывает развёртывание серверной и клиентской части приложения **Prime-Media Messenger** (АО «Прайм-Медиа») и результаты ручного функционального тестирования.

---

## Требования к окружению

### Клиент (Android)

| Параметр | Значение |
|----------|----------|
| ОС | Android **7.0** и выше |
| API level | **24+** (`minSdk 24` в проекте) |
| Устройство | Физическое устройство с USB-отладкой или эмулятор (рекомендуется API 34 для разработки) |

### Сервер

| Параметр | Значение |
|----------|----------|
| Python | **3.8** или выше |
| OpenSSL | Для генерации TLS-сертификатов (`generate_certs.sh`) |
| Сеть | Открытый TCP-порт **12345** в локальной сети |
| ОС | Linux (Ubuntu 20.04+), Windows Server 2019+, macOS |

### Сборка из исходников (опционально)

- **Android Studio** с поддержкой AGP 7.4.2
- **JDK 11**
- Gradle Sync проекта

## Установка сервера

1. Скопируйте каталог `server/` на машину, которая будет принимать подключения клиентов.

2. Установите зависимости Python (если в каталоге есть `requirements.txt`):
   ```bash
   pip install -r requirements.txt 
   ```
   Если файла `requirements.txt` нет, используются стандартные библиотеки Python из `main.py`.

3. Сгенерируйте TLS-сертификаты:
   ```bash
   cd server
   chmod +x generate_certs.sh
   ./generate_certs.sh
   ```
   В каталоге `server/certs/` появятся `cert.pem` и `key.pem`.

4. Запустите сервер в режиме отладки:
   ```bash
   python3 main.py -d
   ```
   Флаг `-d` включает подробное логирование (в том числе в `logs/messenger_YYYYMMDD.log`).

5. При первом запуске создаётся база `server/data/messenger.db` и тестовые пользователи (`user1` / `password1`, `user2` / `password2`, `user3` / `password3`, `admin` / `admin`).

6. Скопируйте `server/certs/cert.pem` в Android-проект как `app/src/main/res/raw/messenger_server_cert` и пересоберите APK, если сертификат менялся.

## Установка клиента (Android)

### Вариант A: готовый APK

1. Скопируйте файл `app-release.apk` на устройство (USB, облако, внутренняя почта).

2. Откройте APK через файловый менеджер.

3. Разрешите установку из **неизвестных источников** (Настройки → Безопасность / Приложения — зависит от производителя).

4. Завершите установку и запустите **Prime-Media Messenger**.

5. Убедитесь, что устройство находится в той же сети, что и сервер, и что в сборке указан корректный IP сервера (см. вариант B, если нужна пересборка).

### Вариант B: настройка IP при пересборке

1. Откройте `app/src/main/res/values/strings.xml`.

2. Измените значение `server_host` на IP-адрес сервера в LAN (для эмулятора на хосте разработки часто используется `10.0.2.2`).

3. Порт задаётся в `app/src/main/res/values/integers.xml` (`server_port`, по умолчанию **12345**).

4. Соберите release- или debug-сборку в Android Studio (см. `DEVELOPER_GUIDE.md`).

### Проверка перед тестами

- Сервер запущен: `python3 main.py -d` в `server/`.
- Сертификат клиента совпадает с серверным.
- На устройстве включены интернет и доступ к порту 12345.
